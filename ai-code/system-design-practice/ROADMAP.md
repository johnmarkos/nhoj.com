# System Design Practice Roadmap

## Now (Active)

1. **Migrate to quizzer-core**
   - When quizzer-core v3.0 ships (extracted from OpenQuizzer — see [OpenQuizzer ROADMAP](https://github.com/johnmarkos/openquizzer/blob/main/ROADMAP.md)), replace the manually-copied `openquizzer.js` with the standalone `quizzer-core.mjs` ES module file. Pin to a release version. No more 3-file-copy upgrade workflow — just update the pinned file. Zero-build-step deployment is preserved (quizzer-core provides a standalone ES module).

2. Content realism calibration
   - Continue definition-to-scenario rewrites in highest-ratio chapters (notably Unit 7, Unit 4, Unit 5, Unit 9/10, Unit 8/12).
   - Continue de-templating remaining chapter clusters where long practice sessions still feel repetitive.

3. Difficulty calibration pass
   - Refine baseline `difficulty` labels against actual interview-style complexity and ambiguity.

4. Interview simulation semantics (deferred engine work)
   - Evaluate strict session-level wall-clock enforcement once quizzer-core bandwidth is available.

## Later

1. Skill-grid refinement
   - Evolve baseline tag bars toward richer proficiency visualization (e.g., Elo-informed charting).

2. Community contributions workflow
   - Define quality gate and contribution process for external problem submissions.

3. Alternative tracks
   - Explore frontend system design, ML systems, and data engineering tracks.

4. Formalized content generation pipeline
   - The existing 9,844 problems were generated by Claude Opus 4.5 and GPT-5.3 Codex through a deliberate generate-lint-review workflow. If expanding into new tracks (item 3), consider formalizing this pipeline into a reusable tool — structured prompts, automated `content-lint.cjs` validation, batch generation. Could also share prompt patterns with PageQuizzer's generation system.

## Notes

- Completed milestones are tracked in `CHANGELOG.md`.
- Baseline validation currently passes with the latest linter and test suite.
- **Ecosystem:** This project is part of the quizzer ecosystem — [OpenQuizzer](https://github.com/johnmarkos/openquizzer) (template engine), [quizzer-core](https://github.com/johnmarkos/quizzer-core) (shared engine package), [PageQuizzer](https://github.com/johnmarkos/page-quizzer) (Chrome extension for LLM-generated quizzes).
